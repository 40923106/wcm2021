Heroku 網站

1. (建立 Heroku 帳號) 進入 https://heroku.com, 利用學校的 @gm 註冊一個 Heroku 帳號

2.(建立 Heroku app) 登入 Heroku 後，先以 s + 學號 建立第一個 app

3. (下載 Heroku CLI) 從 http://a.kmol.info:88/heroku.7z 下載 Heroku cli 程式檔案，解開至隨身碟 Y: 所在位置

4. (修改命令搜尋路徑) 修改隨身系統的啟動批次檔案, 讓 Windows 的命令視窗中可以執行 heroku.exe，重新啟動已經納入可執行 heroku.exe 的隨身碟 start_ipv6.bat 或 start_ipv4.bat中

5.(測試 heroku 執行) 重新打開start_ipv6.bat，在命令列中執行輸入 heroku version，如果有回應表示有設定成功

6.(設定 heroku 代理主機) 在 y:\home_ipv6 下的 .giconfig 中， 以 git config --global http.proxy "http://your_proxy:port" 進行設定

在 start_ipv6.bat 中加入:

1
2
3
REM for heroku login
REM set HTTP_PROXY=http://[2001:288:6004:17::53]:3128
REM set HTTPS_PROXY=http://[2001:288:6004:17::53]:3128


因為是用自己得網路，所以要在前面打REM變成註解

7.(設定同步倉儲) 透過 git remote add (Heroku 指令為 heroku git:remote -a heroku_倉儲_名稱, 之後只要在 heroku login 狀態下, 可以直接利用 git push) 在近端設定一個連線到 Heroku 倉儲的代號, 就能在每次更新 Github Pages 倉儲後, 再將內容同步至 Heroku

8.(以 git 進行提交推送) 只要使用 git push heroku 就可以將改版內容推送到對應的 Heroku app 倉儲

網路直播

接下來要講解如何用 OBS+YouTube 進行直播

1.點選右上角的攝影機按鈕並點選進行直播

2. 如果是第一次進行直播需要驗證後等24小時才能啟用直播

3.輸入標題及其他一些設定，完成後點選繼續

4.將串流設定中的串流金鑰複製

5.進到OBS中的設定，點選串流，服務選YouTube，伺服器選Primary YouTube ingest server，然後將剛剛複製的串流金鑰貼上，點選套用和確定

6.接著點選一般，勾選串流時自動錄製，按下套用及確定

7.接著點選輸出，設定錄影路徑，綠影格式選mp4，好了後點選套用及確定

8.點選開始串流

9.回到YouTube畫面，此時YouTube的直播畫面就會和OBS同步顯示並且進行直播

RoboDk執行

1.我是用自己的網路操作Robodk，所以要先找到ipv4的位置，在命令列打ipconfig /all，即可找到自己ipv4的位置

2.接著把robodk522_protable>example>pick_and_place裡面的pick_and_place.py拉到SciTE裡，在Robolink裡打入自己的ipv4位置

3.打開Robodk.exe，按左上角的資料夾圖案，選取example中除了pick_and_place.py之外的其他檔案後開啟

4.按工具>選項

5.打開選項後，勾取允許外部API

6.點選API服務器IP後，確定網路位置是正確的ipv4位置後按OK

7.回到pick_and_place.py的SciTE中點選Tools Go，即可啟用Robodk
